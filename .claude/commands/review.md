# 블로그/유튜브 분석 리뷰

인자: $ARGUMENTS

수집된 데이터로 블로그 또는 YouTube 채널 분석을 진행합니다.

## 플랫폼 판별

분석 대상이 YouTube인지 블로그인지 자동 판별:
- `output/` 폴더명이 `yt_`로 시작 → **YouTube 모드**
- summary 파일에 `"platform": "youtube"` 있음 → **YouTube 모드**
- 그 외 → **네이버 블로그 모드**

## 인자 파싱 (최우선)

인자가 있으면 단계를 건너뛰고 바로 실행합니다.

**패턴:**
- `<별명/ID> <포스트번호>` → 해당 블로그/채널의 특정 포스트/영상 바로 분석
  - 예: `/review 비움 00` → hhey0510의 00번 포스트 특정글분석
  - 예: `/review 테크몽 00` → yt_techmong의 00번 영상 분석
- `<별명/ID> <수량>` → 해당 블로그/채널의 포스트/영상 N개 목록 표시 후 번호 선택 대기
  - 예: `/review 비움 5` → hhey0510의 포스트 5개 목록 표시
- `<별명/ID> 0` → 해당 블로그/채널 전체분석 바로 실행
  - 예: `/review 비움 0` → hhey0510 전체분석
  - 예: `/review 테크몽 0` → yt_techmong 전체분석
- `<별명/ID> <포스트번호> 댓글` → **댓글만 바로 생성** (분석 생략)
  - 예: `/review 리리 00 댓글` → 00번 포스트 댓글만 출력
- `<별명/ID> 댓글` → 최신 포스트 댓글만 바로 생성
  - 예: `/review 리리 댓글` → 최신 글 댓글만 출력
- `<별명/ID> cross <관점별명/ID>` → **관점 분석** 실행
  - 예: `/review 초코홈 cross 셀슈머` → 초코홈을 셀슈머 관점으로 분석
  - 예: `/review 테크몽 cross 비움 벤치마킹` → 프레임 지정 분석
- `cross list` → 관점 분석 목록 조회
  - 예: `/review cross list` → 전체 관점 분석 목록
  - 예: `/review cross list 셀슈머` → 셀슈머 관점 분석만 필터
- `<별명/ID>` → 해당 블로그/채널 선택 후 분석 유형 선택 대기
- 인자 없음 → 아래 "자동 연결" 로직 실행

**별명/ID 매칭:** `config/blogs.json`에서 ID, name, nickname, description 순으로 검색. 유튜브 채널은 `config/youtubers.json`에서 ID, name, nickname, handle 순으로 검색.

## 자동 연결 (인자 없음일 때)

`/review`만 실행했을 때, 직전 `/search` 결과를 자동으로 이어받습니다.

**판단 기준:** `config/.last_search.json` 파일 확인
- 파일 있음 → 해당 blog_id의 **최신** summary/posts 파일 자동 선택
  - 포스트 1개 → **질문 없이** 바로 특정글분석 실행
  - 포스트 2~5개 → 포스트 목록 표시 후 번호 선택 대기
  - 포스트 6개 이상 → 분석 유형 선택 대기 (0: 전체분석, 또는 포스트 번호)
- 파일 없음 → 계정 목록 표시 (기존 흐름)

## 실행 흐름 (자동 연결 불가 시)

### 1단계: 계정 목록 표시

output 폴더의 계정(폴더)을 번호와 함께 마크다운 표로 출력합니다.
계정 폴더 목록 조회 시 `_old` 등 `_`로 시작하는 폴더는 제외합니다.

| 번호 | 플랫폼 | 계정 |
|------|--------|------|
| 1    | 블로그 | 980207 |
| 2    | 블로그 | shootingday |
| 3    | YouTube | yt_techmong |

그리고 "계정 번호를 입력하세요:" 출력 후 대기.

### 2단계: 분석 유형 선택

사용자가 계정 번호를 입력하면:

```
분석 유형을 선택하세요:
- 0: 전체분석 (12개 항목)
- 1~50: 해당 수량만큼 포스트/영상 목록 표시

입력:
```

그리고 대기.

### 3단계: 분기 처리

**"0" 입력 시 → 전체분석:**
- 해당 계정의 최신 `*_summary_*.json` 파일을 읽습니다
- `_guide`의 "전체분석" 형식(12개 항목)으로 블로그/채널 전체를 분석합니다
- 12개 항목: 콘텐츠장단점, 운영자문제점, SEO개선점, 전략제안, 타겟독자, 브랜딩, 발행패턴, 제목패턴, 수익화, 경쟁환경, AI대응력, 독자참여도

**YouTube 전체분석 시 추가 분석:**
- summary의 `_guide.youtube분석` 항목을 참고하여 추가 분석:
  - 자주 다루는 **제품군/주제 TOP 5** (비율 포함)
  - **조회수 높은 주제 패턴** (평균 조회수 기준 랭킹)
  - **업로드 빈도**, 요일, 시간대
  - **영상 길이 분포** (숏폼 ~1분 / 미들 1~10분 / 롱폼 10분+)
  - **제목 패턴** (낚시성, 키워드, 이모지 사용 여부)
  - **협찬/광고 비율** 추정 (설명란 분석)
  - **태그 활용도**
  - **구독자 대비 조회율**

**"1~50" 입력 시 → 포스트/영상 목록:**
- 해당 계정의 최신 `*_summary_*.json` 파일에서 입력한 수량만큼 목록 표시

블로그:
| 번호 | 제목 | 날짜 |
|------|------|------|
| 00   | 제목1 | 02/10 |

YouTube:
| 번호 | 제목 | 날짜 | 조회수 | 시간 |
|------|------|------|--------|------|
| 00   | 영상제목 | 02/10 | 52만 | 12:30 |

그리고 "분석할 번호를 입력하세요 (예: 00, 01):" 출력 후 대기.

### 4단계: 특정글/영상 분석

포스트 번호가 결정되면:
- 해당 계정의 최신 `*_posts_*.json` 파일에서 해당 index의 본문을 읽습니다
- `_guide`의 "특정글분석" 형식으로 심층 분석합니다
- **블로그**: 8개 섹션 (콘텐츠 구조 분석, 강점 4가지, 약점 및 개선점, SEO 분석, 독자 반응 예측, 종합 평가, 총평, 추천 댓글)
- **YouTube**: 7개 섹션 (콘텐츠 구조 분석, 강점 4가지, 약점 및 개선점, SEO 분석, 시청자 반응 예측, 종합 평가, 총평)
  - YouTube는 자막(content)을 본문으로 사용
  - 조회수/좋아요 통계도 함께 표시
  - 추천 댓글 대신 영상 통계 요약

## 댓글만 생성 모드 (네이버 블로그 전용)

인자에 `댓글`이 포함되면 분석을 생략하고 **추천 댓글만** 바로 출력합니다.

**실행 흐름:**
1. 해당 블로그의 최신 `*_posts_*.json` 파일에서 본문 읽기
2. 본문을 바탕으로 추천 댓글 생성
3. 댓글만 출력 (분석 8개 섹션 생략)

**댓글 작성 가이드:**
- 글의 강점 1~2개 구체적으로 언급
- 공감 표현 (개인 경험 또는 감정)
- 후속 콘텐츠에 대한 기대감
- 자연스러운 어조 (과도한 칭찬 지양)
- 2~4문장 적정 길이

**출력 형식:**
```
## 추천 댓글

---
[댓글 내용 - 인용 블록(>) 없이 평문으로 출력하여 복사하기 쉽게]
---

📝 글 제목: [제목]
🔗 [글 링크](URL)
```

**중요:** 댓글 내용은 `>` 인용 블록 없이 평문으로 출력하여 바로 복사할 수 있게 합니다.

## 관점 분석 (Cross Analysis)

등록된 블로거/유튜버의 **관점(페르소나)**으로 다른 채널을 분석합니다.
크로스 플랫폼 지원: 블로거→유튜버, 유튜버→블로거, 동종 플랫폼 모두 가능.

### 명령어

```
/review <대상> cross <관점>              # 프레임 자동 판별
/review <대상> cross <관점> <프레임>     # 프레임 지정 (비즈니스|벤치마킹|경쟁)
/review cross list                       # 관점 분석 목록
/review cross list <관점별명>            # 특정 관점 필터
```

### 전제조건

| 데이터 | 출처 | 필수 |
|--------|------|------|
| 대상 전체분석 | `docs/reviews/{대상}_전체분석_*.md` | 필수 |
| 관점 제공자 전체분석 | `docs/reviews/{관점}_전체분석_*.md` | 필수 |
| 관점 제공자 페르소나 | `config/personas/{이름}.json` | 선택 |
| 관점 제공자 context_file | 페르소나 JSON의 `context_file` | 선택 |
| 대상/관점 메타데이터 | `config/blogs.json` 또는 `config/youtubers.json` | 필수 |

**전체분석이 없을 때:**
```
"{대상}의 전체분석이 없습니다.
`/review {대상} 0`으로 먼저 전체분석을 실행하세요."
```

### 실행 흐름

```
1. 대상 ID 매칭 (blogs.json + youtubers.json)
2. 관점 제공자 ID 매칭 (blogs.json + youtubers.json)
3. 전제조건 확인:
   - 대상 전체분석 파일 존재? (Glob: docs/reviews/{대상ID}_전체분석_*.md)
   - 관점 제공자 전체분석 파일 존재?
   - 없으면 안내 후 중단
4. 데이터 로드:
   - 대상 전체분석 읽기 (최신 파일)
   - 관점 제공자 전체분석 읽기 (최신 파일)
   - 관점 제공자 페르소나 JSON (있으면)
   - 관점 제공자 context_file (있으면)
5. 분석 프레임 결정:
   - 인자로 지정했으면 → 그대로 사용
   - 미지정 → 자동 판별 후 텍스트로 확인 질문
6. 분석 실행 (프레임별 섹션 순서대로)
7. 파일 저장
8. 결과 출력
```

### 분석 프레임 (3종)

#### 프레임 자동 판별 로직

```
1. 관점 제공자의 전체분석에서 topics, description 확인
2. 대상과 관점 제공자의 주제 비교:
   - topics.main이 같거나 유사 → "경쟁" 추천
   - topics.main이 다르고, 관점 제공자가 비즈니스/마케팅 관련 → "비즈니스" 추천
   - 그 외 → "벤치마킹" 추천
3. 추천 프레임을 텍스트로 표시하고 확인 대기:
   "분석 프레임: {추천} (비즈니스/벤치마킹/경쟁 중 선택, 엔터=추천 적용)"
```

#### A. 비즈니스 파트너 평가

**적합 관점:** 벤더, 마케터, 광고주, 사업가

| # | 섹션 | 내용 |
|---|------|------|
| 1 | 분석 배경 | 관점 제공자 소개 + 분석 목적 |
| 2 | 평가 기준 도출 | 관점 제공자의 글/영상에서 핵심 기준 3가지 추출 |
| 3 | 기준 대입 결과 | 대상에게 기준 적용 → 판정표 |
| 4 | 비즈니스 점수표 | 7~8개 항목 정량 평가 |
| 5 | 매출 시뮬레이션 | 구체적 숫자 시나리오 (일반/중박/대박) |
| 6 | 협업 판단 (Go/No-Go) | 단건/장기/독점 실행 판단 + 최적 제안 상품 |
| 7 | 상호 인사이트 | 관점 제공자가 배울 점 + 아쉬운 점 |
| 8 | 총평 | 한 줄 결론 + 전략 제안 |

**점수표 항목:** 카테고리적합도, 콘텐츠전환력, 바이럴가능성, 가격대비효율, 타겟일치도, 광고투명도, 재협업가능성

#### B. 벤치마킹 분석

**적합 관점:** 동종/유사 분야 크리에이터

| # | 섹션 | 내용 |
|---|------|------|
| 1 | 분석 배경 | 관점 제공자 소개 + 벤치마킹 이유 |
| 2 | 프로필 비교 | 기본 수치 비교 (구독자/포스트수, 주제, 톤) |
| 3 | 콘텐츠 스타일 비교 | 글쓰기/영상 스타일, 구조, 톤앤매너 차이 |
| 4 | 강점 벤치마킹 3가지 | 대상에게 배울 수 있는 구체적 기법 |
| 5 | 적용 가능한 전략 | 관점 제공자가 자기 채널에 적용할 액션 |
| 6 | 차별화 포인트 | 대상과 겹치지 않는 관점 제공자만의 강점 |
| 7 | 실행 액션 플랜 | 당장 시도할 수 있는 3가지 액션 |

#### C. 경쟁 분석

**적합 관점:** 같은 주제/타겟을 다루는 경쟁 채널

| # | 섹션 | 내용 |
|---|------|------|
| 1 | 분석 배경 | 경쟁 관계 설정 이유 |
| 2 | 포지셔닝 비교 | 주제, 타겟, 톤, 플랫폼 매트릭스 |
| 3 | 수치 비교 | 구독자, 조회수, 발행빈도, 성장률 |
| 4 | 콘텐츠 겹침 영역 | 둘 다 다루는 주제와 각자의 차이점 |
| 5 | 상대의 빈 틈 | 관점 제공자가 공략할 미개척 영역 |
| 6 | 경쟁 회피 전략 | 직접 경쟁 대신 차별화 포인트 |
| 7 | 총평 | 경쟁 관계 요약 + 전략 제언 |

### 표준 frontmatter

```yaml
---
type: cross_analysis
target_id: "{대상 ID}"
target_name: "{대상 채널/블로그명}"
target_platform: "youtube|blog"
perspective_id: "{관점 제공자 ID}"
perspective_name: "{관점 제공자 별명}"
perspective_platform: "youtube|blog"
perspective_role: "{역할 설명}"
analysis_frame: "비즈니스|벤치마킹|경쟁"
analyzed_at: "{YYYY-MM-DD}"
score: {종합점수}
verdict: "{한 줄 판정}"
tags: [{태그들}]
---
```

**비즈니스 추가 필드:**
```yaml
partner_score: 78
단건협업: "YES|NO|조건부"
장기파트너: "YES|NO|조건부"
독점계약: "YES|NO|조건부"
최적_카테고리: ["카테고리1", "카테고리2"]
```

**벤치마킹 추가 필드:**
```yaml
benchmark_score: 85
top_learning: "30초 숏폼 구조"
applicable_actions: 3
```

**경쟁 추가 필드:**
```yaml
overlap_ratio: "40%"
competitive_advantage: "기술 깊이"
recommended_strategy: "차별화"
```

### 파일 저장

**파일명:** `{대상ID}_{관점별명}관점_{프레임}_{YYYYMMDD}.md`
**저장 위치:** `docs/reviews/`

예시:
- `yt_chocohomee_셀슈머관점_비즈니스_20260224.md`
- `yt_techmong_비움관점_벤치마킹_20260226.md`
- `zeroenter_테크몽관점_경쟁_20260226.md`

### 관점 분석 목록 (`cross list`)

```
1. docs/reviews/ 에서 `*관점*` 패턴 Glob
2. 각 파일 frontmatter에서 메타데이터 추출 (첫 20줄)
3. 테이블 출력
```

**출력 형식:**
```
## 관점 분석 목록 (N개)

| # | 대상 | 관점 | 프레임 | 점수 | 날짜 |
|---|------|------|--------|------|------|
| 1 | 초코홈 | 셀슈머 | 비즈니스 | 78 | 2026-02-24 |

---
💡 `/review cross list 셀슈머` - 특정 관점 필터
💡 `/review <대상> cross <관점>` - 새 관점 분석
```

---

## 중요 사항

- **AskUserQuestion 도구를 사용하지 마세요.** 텍스트로 질문하고 사용자 입력을 기다립니다.
- 사용자 입력이 필요한 단계에서만 대기합니다.
- **자동 연결이 가능하면 질문 없이 바로 분석을 시작합니다.**
