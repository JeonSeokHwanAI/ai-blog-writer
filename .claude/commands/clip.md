# 블로그 글 클리핑

인자: $ARGUMENTS

읽어온 블로그 글을 참고용 md 파일로 저장합니다.

## 사용법 분기

**인자 분석:**
- 인자 없음 → 최근 읽은 블로그 글 저장
- `list` → 저장된 클립 목록 보기
- `<번호>` → 해당 클립을 VSCode에서 열기
- `#<태그>` → 태그로 필터링
- `tag <번호> <태그들>` → 클립에 태그 추가
- `untag <번호> <태그>` → 클립에서 태그 제거

## 성능 최적화

**핵심 원칙:** 모든 작업은 이 대화창 안에서 완료. 사용자가 직접 폴더를 열 필요 없음.

1. **목록 조회 (`list`)**:
   - Glob으로 파일 목록만 빠르게 조회
   - 각 파일의 frontmatter에서 필요한 정보만 추출 (첫 25줄)
   - 테이블 + 번호 안내

2. **파일 열기 (`<번호>`)**:
   - `code` 명령으로 VSCode에서 직접 열기
   - 명령: `code "파일경로"`

3. **인덱스 캐싱**:
   - 클립 저장 시 `config/.clips_index.json`에 메타데이터 저장
   - list 호출 시 인덱스 파일 먼저 확인 (없으면 생성)

## 실행 방법

### 1. 클립 저장 (인자 없음)

**전제조건:** 대화에서 블로그 글을 읽어온 상태여야 함

**저장 흐름:**
1. 현재 대화에서 읽은 블로그 정보 확인
2. AskUserQuestion으로 질문:
   - 용도: 참고 / 인용 / 벤치마킹 / 영감
   - 분류: 콘텐츠기획 / 글쓰기 / 마케팅 / 기타
3. **AI 태그 자동 생성**: 본문 분석하여 태그 3~5개 자동 추출
4. 파일 생성: `docs/clips/{제목_sanitized}.md`
5. 인덱스 업데이트: `config/.clips_index.json`

**AI 태그 생성 규칙:**
- 본문에서 핵심 키워드 3~5개 추출
- 태그 유형:
  - **주제 태그**: 글의 메인 주제 (예: SEO, 제목작성, 스토리텔링)
  - **기법 태그**: 글쓰기 기법 (예: 후킹, CTA, 구조화)
  - **감정 태그**: 글의 톤/분위기 (예: 공감형, 정보형, 설득형)
- 한글 태그 우선, 영어는 널리 쓰이는 경우만 (SEO, CTA 등)
- 너무 일반적인 태그 제외 (블로그, 글쓰기 등)

**저장 완료 후 출력:**
```
클립 저장 완료: {제목}

| 항목 | 내용 |
|------|------|
| 파일 | docs/clips/{파일명}.md |
| 분류 | {분류} |
| 용도 | {용도} |
| 태그 | {자동생성된 태그들} |

---
💡 `/clip tag {번호} 추가태그` - 태그 추가
💡 `/clip untag {번호} 제거할태그` - 태그 제거
💡 `/clip list` - 전체 목록 보기
```

**파일 구조:**
```markdown
---
source: naver_blog
blog_id: "{blog_id}"
post_id: "{post_id}"
url: "{url}"
author: "{author}"
title: "{title}"
created_at: "{created_at}"
clipped_at: "{clipped_at}"
purpose: "{용도}"
category: "{분류}"
tags: [{태그들}]
---

# {title}

> 원본: {url}
> 작성자: {author}
> 작성일: {created_at}

---

{본문 내용}
```

### 2. 클립 목록 보기 (`list`)

**실행 순서:**
1. `config/.clips_index.json` 확인
2. 없으면 Glob + Read로 인덱스 생성
3. 테이블 출력

**출력 형식:**
```
## 저장된 클립 (N개)

| # | 제목 | 작성자 | 분류 | 용도 | 저장일 |
|---|------|--------|------|------|--------|
| 1 | 제목1 | 작성자1 | 글쓰기 | 참고 | 2026-02-11 |

---
`/clip 1` - 파일 열기
`/clip #글쓰기` - 태그 필터
```

### 3. 클립 열기 (`<번호>`)

해당 번호의 클립 파일을 VSCode에서 직접 엽니다.

**실행:**
```bash
code "docs/clips/{파일명}.md"
```

**출력:**
```
클립 #{번호} 열었습니다: {제목}
```

### 4. 태그 필터링 (`#<태그>`)

인덱스에서 해당 태그가 있는 클립만 필터링하여 목록 표시.

### 5. 태그 추가 (`tag <번호> <태그들>`)

클립에 태그를 추가합니다.

**실행 방법:**
1. `config/.clips_index.json`에서 해당 번호의 클립 찾기
2. 클립 파일(`docs/clips/{파일명}.md`)의 frontmatter에서 tags 업데이트
3. 인덱스 파일도 함께 업데이트

**태그 형식:**
- 쉼표로 구분: `태그1, 태그2, 태그3`
- 공백으로 구분: `태그1 태그2 태그3`
- 혼합 가능

**예시:**
```
/clip tag 1 SEO, 제목작성법
/clip tag 2 글쓰기팁 구조화
```

**출력:**
```
클립 #1에 태그 추가됨: SEO, 제목작성법

| 항목 | 내용 |
|------|------|
| 제목 | {제목} |
| 기존 태그 | {기존태그} |
| 추가된 태그 | SEO, 제목작성법 |
| 전체 태그 | {전체태그} |
```

### 6. 태그 제거 (`untag <번호> <태그>`)

클립에서 특정 태그를 제거합니다.

**예시:**
```
/clip untag 1 SEO
```

**출력:**
```
클립 #1에서 태그 제거됨: SEO
남은 태그: {남은태그}
```

## 인덱스 파일 구조

`config/.clips_index.json`:
```json
{
  "clips": [
    {
      "id": 1,
      "filename": "톡방으로_전달되는_영상의_가치.md",
      "title": "톡방으로 전달되는 영상의 가치",
      "author": "띵큐",
      "category": "글쓰기",
      "purpose": "영감",
      "tags": ["스토리텔링", "영상콘텐츠", "공감형"],
      "clipped_at": "2026-02-11"
    }
  ],
  "updated_at": "2026-02-12T16:30:00"
}
```

## 참조 파일
- `docs/clips/`: 클립 저장 폴더
