## 1. 프로젝트 소개

### 이 프로젝트는 무엇인가요?

AI를 파트너로 활용하여 나만의 블로그 작가가 되는 플랫폼입니다.
블로그 분석부터 페르소나 기반 글쓰기까지, AI 블로그 작가가 직접 만든 도구로 성장 전략을 세워보세요.

> AI는 펜이고, 당신이 작가입니다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| 블로그 스크래핑 | 네이버 블로그 포스트 자동 수집 |
| 전체 분석 | 12개 항목으로 블로그 전체 평가 |
| 개별 글 분석 | 특정 포스트 심층 리뷰 |
| 키워드 분석 | 검색 키워드 트렌드 파악 |
| 페르소나 생성 | 인터뷰 기반 나만의 글쓰기 스타일 생성 |
| 글쓰기 지원 | 페르소나 기반 블로그 글 작성 |

## 2. 환경 설정

### 필수 요구사항

- Python 3.10 이상
- Claude Code (VS Code 확장)

### 2.1 Python 설치

Claude Code에게 다음과 같이 요청하세요:

```
Python 3.12를 설치해줘
```

또는 직접 설치:
1. https://www.python.org/downloads/ 접속
2. Python 3.12 다운로드
3. 설치 시 **"Add Python to PATH"** 체크 필수!

**설치 확인:**
```bash
python --version
# Python 3.12.x 출력되면 성공
```

### 2.2 의존성 설치

Claude Code에게 다음과 같이 요청하세요:

```
requirements.txt 파일을 읽고 필요한 패키지를 설치해줘
```

또는 직접 설치:
```bash
# 프로젝트 폴더로 이동
cd "e:/Project/vibecode/Naver Scraping"

# 의존성 설치
pip install -r requirements.txt
```

**설치 확인:**
```bash
pip list
# requests, beautifulsoup4 등 패키지 목록 출력되면 성공
```

### 2.3 Claude Code 설치

1. VS Code 실행
2. 확장(Extensions) 탭 열기 (Ctrl+Shift+X)
3. "Claude Code" 검색
4. 설치 버튼 클릭

## 3. 기본 사용법

### 3.1 블로거 목록 확인

```
/bloggers
```

등록된 블로거 목록을 확인합니다.

### 3.2 블로그 스크래핑

```
/search <블로그ID> <수량>
```

**예시:**
- `/search zeroenter 5` - zeroenter 블로그 최신 5개 포스트 수집
- `/search 리리 3` - 별명으로 검색 (자동 매칭)

### 3.3 블로그 분석

```
/review
```

수집된 데이터로 분석 진행:
- **0**: 전체 분석 (12개 항목)
- **1~50**: 해당 수량만큼 포스트 목록 표시 후 개별 분석

### 3.4 댓글 빠르게 보기

분석 없이 추천 댓글만 바로 생성합니다.

```
/review 댓글
```

> 직전 스크래핑한 블로그의 최신 글에 대한 댓글 생성

```
/review 비움 댓글
```

> 특정 블로거의 최신 글에 대한 댓글 생성

```
/review 02 댓글
```

> 직전 스크래핑 결과에서 02번 포스트에 대한 댓글 생성

**활용 팁:** 이웃 블로거 글에 빠르게 댓글을 달고 싶을 때 유용합니다.

## 4. 실습 과정

### Step 1: 내 블로그 등록하기

#### 1-1. bloggers 기능 보기
```
/bloggers
```

#### 1-2. 네이버 에서 blogger 찾기

```
/bloggers ?핌포나
/bloggers ?pimpona
/bloggers ?@zziba -> 블로그ID로 검색
```

#### 1-3. blog 게시글 검색
```
/search zziba 10 -> 최근 게시글 부터 10개 포스트 수집
```

#### 1-4. blog 게시글 분석
```
/review 0 -> 수집한 10개 포스트 분석하기
```

#### 1-5. 블로거 추가
```
/bloggers add <내블로그ID> ex)zziba
```

#### 1-6. 등록된 blogger 찾기
```
/bloggers 핌

```

#### 1-7. 등록된 blogger 목록
```
/bloggers all
```

#### 1-8. 블로그 정보 업데이트를 위해 다시 검색
```
/search zziba 30
```

```
/review 0 -> 다시 전체 분석
```

#### 1-9. blogger 정보 업데이트
```
/bloggers update zziba
```

#### 1-10. 개별 정보 업데이트 -> 자연어로 입력
```
zziba 블로거 별명은 핌포나 입니다. 수정하세요
```

#### 1-11. blogger 정보 변경 이력 확인
```
/bloggers history zziba
```

#### 1-12. 등록된 blogger 삭제
```
/bloggers del zziba
```
```
/bloggers add zziba -> 다시 추가 (단, 프로그램 종료하면 안됩니다.)
```

### Step 2: 내 블로그 스크래핑

#### 2-1. 블로그 글 검색하기
```
/search <내블로그ID> 20
```
> **20**: 최신 포스트 20개를 수집한다는 의미입니다. 전체 분석을 위해 최소 10~20개 권장.


#### 2-2. 포스팅 주소로 검색하기
```
/search ?https://blog.naver.com/zziba/224176460764
```

#### 2-3 포스팅 주소로 댓글 만들기
```
/search ?https://blog.naver.com/zziba/224176460764
```

#### 2-4 별명으로 가장 최신글 댓글 만들기
```
/search 핌포나 댓글
```


### Step 3: 분석 받기 

review는 Search 기능 이후에 진행합니다.

#### 3-1. 전체분석하기 (12개 섹션)
```
/review <내블로그ID> 0
```

> **0**: 전체 분석(12개 항목)을 의미합니다.

**12개 항목:**
콘텐츠장단점, 운영자문제점, SEO개선점, 전략제안, 타겟독자, 브랜딩, 발행패턴, 제목패턴, 수익화, 경쟁환경, AI대응력, 독자참여도

#### 3-2. 특정글 분석 받기 (8개 섹션)

```
/review <내블로그ID> 10
```

> **10**: 최신 포스트 10개 목록을 표시합니다. 목록에서 번호를 선택하면 해당 글을 8개 섹션으로 심층 분석합니다.

**8개 섹션:**
콘텐츠 구조 분석, 강점 4가지, 약점 및 개선점, SEO 분석, 독자 반응 예측, 종합 평가, 총평, 추천 댓글


### Step 4: 분석 결과 저장 및 검색(키워드 중심)

#### 분석결과 저장
```
/save
```

#### 분석결과 검색
```

/save keywords 나노바나나

/save list 약점

/save list 핌포나 나노바나나 약점

/save list 강점
```

### Step 5: 결과 확인

`docs/reviews/` 폴더에서 분석 리포트 확인



### Step 6: 좋은 글 클리핑 (선택)

다른 블로거의 좋은 글을 발견했다면 참고용으로 저장할 수 있습니다.

#### 6-1. 클리핑할 블로그 검색

```

https://blog.naver.com/questioncue/224182192803

/bloggers ?https://blog.naver.com/questioncue/224182192803

```

#### 6-2. 블로그 글 불러오기

```
/search ?https://blog.naver.com/questioncue/224182192803
```

> 검색된 블로그 ID(`questioncue`)로 글을 불러옵니다.

#### 6-3. 특정 글 분석 후 클리핑

```
/review ?https://blog.naver.com/questioncue/224182192803
```

> 포스트 목록에서 번호(예: 00)를 선택하여 글 내용을 확인합니다.

```
/clip ?https://blog.naver.com/questioncue/224182192803
```

> 클리핑 실행 시 다음을 선택합니다:

| 선택 항목 | 옵션 | 설명 |
|-----------|------|------|
| **용도** | 참고 | 나중에 참고할 자료 |
| | 인용 | 글 작성 시 인용할 내용 |
| | 벤치마킹 | 글쓰기 스타일 벤치마킹 |
| | 영감 | 아이디어/영감 얻기 |
| **분류** | 콘텐츠기획 | 콘텐츠 기획 관련 |
| | 글쓰기 | 글쓰기 기법/스타일 |
| | 마케팅 | 마케팅/홍보 관련 |
| | 기타 | 그 외 분류 |

> 저장 위치: `docs/clips/{제목}.md`

**실제 저장 예시:**
```
docs/clips/톡방으로_전달되는_영상의_가치.md
```

| 항목 | 내용 |
|------|------|
| 블로그 ID | questioncue |
| 작성자 | 띵큐 |
| 제목 | 톡방으로 전달되는 영상의 가치 |
| 용도 | 영감 |
| 분류 | 글쓰기 |

#### 6-4. 클리핑 목록 확인

```
/clip list
```

> 저장된 클립 목록을 표로 확인합니다.

#### 6-5. 태그로 필터링

```
/clip #글쓰기
```

> 특정 태그가 붙은 클립만 필터링합니다.

**클리핑 활용 예시:**
- **벤치마킹**: 잘 쓴 제목 패턴, 글 구조, CTA 방식 참고
- **인용**: 통계, 사례, 전문가 의견 등 출처 자료 보관
- **영감**: 새로운 주제 아이디어, 표현 방식 수집

## 5. AI로 글쓰기 연습

### 5.1 트렌드 검색

먼저 트렌드를 파악하여 어떤 주제가 인기인지 확인합니다.

```
/trend 비즈니스·경제
```

> 분야별 핫키워드와 트렌드 분석 결과를 보여줍니다.

### 5.2 키워드 분석

트렌드에서 발견한 키워드를 상세 분석합니다.

```
/keyword AI 바이브코딩
```

> 연관 키워드, 검색 의도, 콘텐츠 추천을 제공합니다.

### 5.3 페르소나 설정

내 블로그 스타일을 AI에게 학습시켜 페르소나를 생성합니다.

#### 5.3.1 현재 페르소나 확인

```
/persona
```

> 현재 활성화된 페르소나 정보를 확인합니다.

**출력 예시:**

| 항목 | 내용 |
|------|------|
| 닉네임 | AI 블로그 작가 |
| 블로그 | zeroenter (지식의 대장장이) |
| 톤 | 습니다체 기본 + 요체 포인트 |
| 구조 | 도입(경험) → 본문(단계별) → 마무리(격려) |

#### 5.3.2 페르소나 목록

```
/persona list
```

> 등록된 모든 페르소나 목록을 확인합니다.

#### 5.3.3 블로그 분석으로 페르소나 생성

```
/persona <내블로그ID>
```

```
/persona style <내블로그ID>
```

> 기존 글을 분석하여 톤앤매너, 문체, 주제를 파악하고 페르소나로 등록합니다.

**분석 항목:**
- 글쓰기 톤 (격식체/반말체/혼합)
- 자주 쓰는 표현
- 문장 구조 패턴
- 이모지/특수문자 사용 빈도
- 평균 글 길이
- 콘텐츠 구조

#### 5.3.4 인터뷰 방식으로 페르소나 생성

```
/persona create <이름>
```

> 블로그가 없어도 AI 인터뷰를 통해 페르소나를 만들 수 있습니다.
> 8개 섹터의 질문에 답하면 나만의 페르소나가 생성됩니다.

**인터뷰 섹터:**
1. 기본 정보 (활동 영역, 전문 분야)
2. 핵심 정체성 (독자에게 어떤 존재?)
3. 콘텐츠 스타일 (선호하는 톤, 소재)
4. 타겟 독자 (주요 독자층, 고민)
5. 콘텐츠 주제 영역
6. 콘텐츠 목표와 가치
7. 독자와의 관계 설정
8. 차별화 포인트

#### 5.3.5 페르소나 전환 (빙의)

```
/persona use <이름>
```

> 등록된 페르소나로 전환합니다. 이후 `/write` 실행 시 해당 스타일로 글을 작성합니다.

#### 5.3.6 페르소나 수정/삭제

```
/persona update <이름>   # 페르소나 수정
/persona delete <이름>   # 페르소나 삭제
```

#### 5.3.7 persona 파일 불러오기
```
persona import @파일명
```

#### 5.3.8 스타일 요약

```
/persona style
```

#### 5.3.

> 현재 활성 페르소나의 글쓰기 스타일만 간단히 확인합니다.

### 5.4 글쓰기 시작

페르소나가 설정된 상태에서 글을 작성합니다.

```
/write
```

```
/write 나노바나나와 포토샵의 차이점
```

Gemini 나노바나나를 활용하여 AI 이미지를 생성할 수 있습니다. 
포토샵은 이미 만들어져 있는 이미지를 가지고 AI로 수정을 할 수 있습니다.

그럼 이 둘의 차이점에 대해 알려주세요. 장단점에 대해서도 그리고 나노바나나로 포토샵 AI를 활용하는 방법에 대해서도


> AI가 내 스타일로 초안을 작성하고, 제목/태그/이미지 프롬프트를 추천합니다.

### 5.5 브릿지 콘텐츠

나의 주제와 관련 없는 키워드를 연결하는 아이디어를 생성합니다.

```
/bridge 다이어트
```

> 예: IT 블로거가 "다이어트" 키워드로 글을 쓰고 싶을 때 → "다이어트 앱 자동화", "칼로리 계산 AI 활용" 등 연결 아이디어 제안

## 7. Q&A

### Q. 스크래핑이 안 돼요

A. 네이버 블로그 URL 형식 확인:
- 올바른 형식: `https://blog.naver.com/블로그ID`
- 블로그 ID만 입력해도 자동으로 URL 생성됩니다

### Q. 분석 결과는 어디에 저장되나요?

A. 두 곳에 저장됩니다:
- 스크래핑 결과: `output/<블로그ID>/`
- 분석 리뷰: `docs/reviews/`

### Q. 내 블로그를 등록하려면?

A. `/bloggers add <블로그ID>` 명령으로 등록 가능합니다.

## 9. 프리미엄 기능 (챌린지 / 더블업)

> 아래 기능들은 **챌린지** 또는 **더블업** 과정에서 제공됩니다.

### 9.1 성장 분석

블로그의 성장을 추적하고 이전 분석과 비교합니다.

```
/analyze <블로그ID>
```

**제공 정보:**
- 12가지 항목 점수 변화 (+/- 표시)
- 발행 패턴 변화
- 주제 비중 변화
- 성장 포인트 / 개선 필요 포인트

**샘플:** [성장분석 샘플](성장분석_샘플.html)

> 컨설팅급 분석 리포트를 원한다면 이 기능을 활용하세요.

### 9.2 HTML 리포트

고객용 분석 리포트를 HTML 랜딩 페이지로 생성합니다.

```
/report <블로그ID>
```

**포함 내용:**
- 종합 점수 대시보드
- 12가지 항목 점수 차트
- 추천 액션 체크리스트
- AI 이미지 생성용 프롬프트

**샘플 리포트:**
- [고객분석 샘플](고객분석_샘플.html)
- [셀슈머 블로그 분석 리포트](../reports/happy_selsumer_report_20260212.html)
- [행복한비움 블로그 분석 리포트](../reports/hhey0510_report_20260212.html)

> 고객에게 납품할 수 있는 수준의 리포트를 생성합니다.

### 9.3 브릿지 콘텐츠

나의 주제와 관련 없는 트렌드 키워드를 연결하는 아이디어를 생성합니다.

```
/bridge <키워드>
```

**5가지 패턴:**
- 도구적용
- 자동화
- 비전공자
- 데이터분석
- 문제해결

> 새로운 독자층을 유입하고 싶을 때 활용하세요.

### 9.4 트렌드 분석

골든키워드를 발굴하고 트렌드를 선점합니다.

```
/trend <분야>
```

**골든키워드 기준:**

| 경쟁강도 | 평가 |
|----------|------|
| < 0.5 | 골든 (검색량 > 문서수 2배) |
| < 1.0 | 좋음 (검색량 > 문서수) |
| 1.0~3.0 | 진입가능 |
| > 3.0 | 레드오션 |

> 돈이 되는 키워드를 찾고 싶다면 이 기능을 활용하세요.

```

"네이버, 카카오, 그립 3개 플랫폼을 직접 사용해본 경험 기반으로 작성.
초보 셀러 관점에서 각 플랫폼 장단점 비교. 수수료 정보 포함."
```

**Step 5: 완성**
```
/write가 자동으로 생성:
- 본문 초안
- 추천 제목 3개
- 태그 10개
- AI 이미지 프롬프트
```

> 💡 **팁**: `/write` 실행 전에 `/persona`로 내 글쓰기 스타일을 등록하면 더 나다운 글이 완성됩니다.

### 9.7 학습형 글쓰기 (프리미엄 전용)

글을 작성할수록 AI가 이전 글들을 기억하고 자동으로 연결해주는 기능입니다.

**핵심 개념:**
- 페르소나별로 작성한 글이 자동 인덱싱
- `/write` 실행 시 관련 글 자동 발견
- 이전 콘텐츠 자동 인용/참조

**작동 방식:**

```
┌─────────────────────────────────────────────────────────────┐
│  1. 글 작성 시 자동 인덱싱                                  │
│     docs/blog/포토샵_AI_20260115.md                         │
│     → config/.posts_index.json에 메타데이터 저장            │
│       - 페르소나: AI블로그작가                              │
│       - 키워드: 포토샵, AI, 이미지편집                      │
│       - 주제: IT·컴퓨터                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  2. /write "나노바나나 AI" 실행 시                          │
│     → 인덱스에서 관련 글 자동 검색                          │
│     → "포토샵 AI 글에서 AI 이미지 생성을 다뤘습니다"        │
│     → 관련 내용 자동 참조 제안                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 자동 인용 생성                                          │
│     "이전 글에서 포토샵 AI로 이미지를 편집하는 방법을       │
│      소개했는데, 오늘은 나노바나나로 이미지를 생성하는      │
│      방법을 알아보겠습니다."                                │
└─────────────────────────────────────────────────────────────┘
```

**인덱스 파일 구조 (`config/.posts_index.json`):**

```json
{
  "AI블로그작가": [
    {
      "file": "docs/blog/포토샵_AI_20260115.md",
      "title": "포토샵 AI로 이미지 편집하기",
      "keywords": ["포토샵", "AI", "이미지편집"],
      "topics": ["IT·컴퓨터"],
      "created_at": "2026-01-15"
    },
    {
      "file": "docs/blog/나노바나나_20260220.md",
      "title": "나노바나나로 AI 이미지 생성하기",
      "keywords": ["나노바나나", "AI", "이미지생성"],
      "topics": ["IT·컴퓨터"],
      "created_at": "2026-02-20"
    }
  ]
}
```

**활용 시나리오:**

| 상황 | AI 동작 |
|------|---------|
| 같은 주제로 새 글 작성 | "이전에 작성한 관련 글이 있습니다" 알림 |
| 관련 키워드 등장 | "이전 글 내용을 인용할까요?" 제안 |
